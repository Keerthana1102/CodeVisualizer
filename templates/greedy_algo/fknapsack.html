Fractional Knapsack Problem: Greedy Approach

Readme

Implementation

Given weights and values of n items, we need to put these items in a knapsack of capacity W to get the maximum total value in the knapsack.

  1. A brute-force solution would be to try all possible subset with all different fraction but that will be too much time taking.
  2. An efficient solution is to use Greedy approach. The basic idea of the greedy approach is to calculate the ratio value/weight for each item and sort the item on basis of this ratio. Then take the item with the highest ratio and add them until we canâ€™t add the next item as a whole and at the end add the next item as much as we can. Which will always be the optimal solution to this problem.

Complexity:
Time Complexity = time for sorting + time for maximizing profit
= time for sorting + O(N)

getMaxValue(wt, val, capacity):
  iVal.sort(reverse=True)
  totalValue = 0
  for i in iVal:
   curWt = int(i.wt)
   curVal = int(i.val)
   if capacity - curWt >= 0:
    capacity -= curWt
    totalValue += curVal
   else:
    fraction = capacity / curWt
    totalValue += curVal * fraction
    capacity = int(capacity - (curWt * fraction))
    break
  return totalValue
