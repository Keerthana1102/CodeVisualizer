Integer Partition Problem

Readme

Implementation

Given a positive integer n, count all possible unique ways to represent n as sum of positive integers.

  1. If we solve this problem recursively, by breaking down the problem into sub-problems, then there exists overlapping sub-structures that makes the time Complexity exponential.
  2. An efficient solution is to use Dynamic Programming Approach. The basic idea is we create a table where each column represents the larger number we have available, and each row represents a possible sum we want to reach. Then we initialize it with the base cases (i.e., mark a 0 on the column that represents the largest number being 0, and a 1 on the first row, which represents the sum being equal to 0). After that we simply need to traverse the table, where each position table[i][j] will be equal to the sum of table[i][j-1] (the part where we donâ€™t take the largest number) and table[i-j][j] (the part where we take the largest number, so the sum will be reduced by the size of the number we just took).
